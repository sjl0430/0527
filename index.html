Routing - 웹 애플리케이션에서 라우팅은 사용자가 요청한 URL에 따라 알맞는 페이지를 보여주는 것을 의미 - 프로젝트를 하나의 페이지(SPA)가 아닌 여러 페이지(MPA)로도 구성 가능 - 페이지 별로 컴포넌트들을 분리해가면서 프로젝트를 관리 SPA - 싱글 페이지 애플리케이션 - HTML을 한 번만 받아와서 웹 애플리케이션을 실행시킨 후, 그 이후에는 필요한 데이터만 받아와서 화면에 업데이트 해주는 것 - 사용자 인터랙션(User Interaction, 사용자와 상호작용)이 별로 없는 정적인 페이지에 적합 - 사용자 인터랙션이 많고 다양한 정보를 제공하는 모던 웹 애플리케이션은 MPA(멀티 페이지 애플리케이션)가 적합 URL(Uniform Resource Locator) - Github pages - 깃허브 사용자가 공개 레퍼지토리에서 호스팅을 할 수 있게 함 쿠키 - 로컬에 저장되는 작은 데이터 파일 - 사용자가 따로 요청하지 않아도 브라우저가 자동으로 HTTP HEADER에 포함하여 서버에 전송 - 서버가 아닌 클라이언트에서 저장되므로, 서버의 부하를 줄일 수 있음 - 보안성이 낮아, 민감한 데이터를 저장하기에 부적합 세션 - 쿠키를 기반으로 하지만, 브라우저가 아닌 서버에서 관리하는 데이터 파일 - 클라이언트와 서버 간 연결이 지속되는 동안 사용자의 상태를 유지 - 데이터를 직접 접근하고 수정하기 어려워 보안성이 높음 - 서버에 저장되므로 대용량 데이터 저장 가능 - 서버에 저장되므로 서버의 부하에 영향을 줄 수 있음 - 사용자가 브라우저를 변경하거나 다른 기기로 이동하면 유지되지 않음 Stateful - client와 server의 통신관계에서 서버가 클라이언트의 상태를 보존 - 즉, 서버가 로그인한 사용자의 상태를 기억함 Stateless - 서버가 클라이언트의 상태를 보존하지 않음 - 토큰을 활용하면 Stateless에서도 사용자 인증할 수 있음 JWT - 클라이언트가 사용자 인증에 필요한 암호화된 로그인 정보들을 토큰에 보관하다가 서버와 통신할 때, 이 토큰을 전달하여 로그인된 사용자임을 인증하는 방식
